# Habit Stacker page — developer overview

This document summarizes **user-facing behavior**, **DOM IDs** (including those created by JavaScript), **assets and scripts**, and **important functions** for the Habit Stacker app (`habit_stacker.html`).

---

## 1. Key user functionality and journeys

### Auth gate

- Unauthenticated users see `#login-required-message` and can use **Log In** (`handleLogin` from shared site/auth scripts loaded with the layout).
- After Auth0 succeeds, `#private-content` is shown and `habits.js` can load the user’s habits.

### Habits list

- The page fetches habits from **`GET /api/habits`**, filters rows where `user_id` matches the logged-in user’s email, sorts by `event_date` (newest first), and renders into `#tasks-tbody`.
- Each row shows **habit name**, **days kept** (whole days from start date through today), and **start date** (formatted).
- Row **background color** reflects streak length: 365+ (black), 100+ (dark green), 21+ (green), 1+ (light green), else white.

### Add habit

- User fills `#task-form`: `#task-input` (name) and `#event-date` (start date).
- Submit runs `addTaskForm` → **`POST /api/habits`** with `task`, `event_date`, and `user_id` (Auth0 email / nickname / sub).
- On success, the form clears and `loadTasks` refreshes the table.

### Row actions (edit / delete)

- Clicking a **data row** (not a button) sets `rowActionHabitId` from `data-habit-id` and opens **`#rowActionModal`** with the habit name in `#row-action-habit-name`.
- **Edit** → closes row modal, calls `editTask(id)` which loads the habit from the API list, fills **`#editEventModal`** (`#edit-task-id`, `#edit-task-input`, `#edit-event-date`), shows the modal.
- **Save** in edit modal → `saveEditTask()` → **`PUT /api/habits/:id`** with `task`, `event_date`, `user_id`.
- **Delete** → `deleteTask(id)` → **`DELETE /api/habits/:id`** after confirm.

### Typical journeys

| Journey | What happens |
|--------|----------------|
| First visit logged out | `updateContentVisibility(false)` → login message; no habits fetch. |
| After login | `habits.js` wraps `updateContentVisibility` → `setTimeout(loadTasks, 100)` when authenticated. |
| Empty list | `#tasks-table` hidden; `#tasks-empty` shows “No habits yet…” (or login prompt if no email). |
| Add first habit | POST → `loadTasks` → table appears with one streak-styled row. |
| Edit habit | Row click → row action modal → edit modal → PUT → `loadTasks`. |
| Delete habit | Row click → delete → confirm → DELETE → `loadTasks`. |

## 2. Important IDs: layout, table, and JS-generated rows

### Page shell

| ID | Role |
|----|------|
| `#private-content` | Authenticated content wrapper (toggled by `auth_page_load.js`). |
| `#login-required-message` | Shown when not logged in. |

### Habits section

| ID | Role |
|----|------|
| `#habits-section` | Card wrapping the habits UI. |
| `#tasks-loading` | Shown while `loadTasks` is fetching. |
| `#tasks-error` | Error alert text on failure. |
| `#tasks-empty` | Info alert when there are no rows (or not logged in). |
| `#tasks-table` | Table hidden until there is at least one habit; then `display: table`. |
| `#tasks-tbody` | **Target for dynamically inserted rows** (`loadTasks`). |

### Add form

| ID | Role |
|----|------|
| `#task-form` | Add habit form. |
| `#task-input` | Habit name. |
| `#event-date` | Start date (`<input type="date">`). |

### Modals

| ID | Role |
|----|------|
| `#rowActionModal` | Choose Edit or Delete after row click. |
| `#row-action-habit-name` | Title text: habit name from clicked row. |
| `#row-action-edit-btn`, `#row-action-delete-btn` | Actions. |
| `#editEventModal` | Edit habit name and start date. |
| `#edit-task-form` | Form wrapper (save via button `onclick="saveEditTask()"`). |
| `#edit-task-id` | Hidden: habit primary key from API. |
| `#edit-task-input` | Habit text. |
| `#edit-event-date` | Start date. |

### Table rows — **generated by `loadTasks()`** (`habits.js`)

For each habit object from the API:

| Attribute / content | Details |
|---------------------|--------|
| `<tr data-habit-id="…">` | `data-habit-id` = numeric `task.id` from API (used for click delegation and edit/delete). |
| **No `id` on `<td>` cells** | Three columns: habit name (`<strong>`), days kept, start date — all text; name is escaped with `escapeHtml(task.task)`. |
| **Inline `style` on `<tr>`** | `backgroundColor` / `color` from streak tier (365 / 100 / 21 / 1 / 0 days). |

Click handling uses `tr[data-habit-id]` and reads the habit name from `td:first-child`.

## 3. Files and important functions

### Assets loaded by the page

| Asset | Role |
|-------|------|
| Layout partials | Navbar, footer (`habit_stacker.html` template). |
| `/habits/habits.css` | Page styles. |
| `/habits/auth_page_load.js` | Auth0 gate, base `updateContentVisibility`, `waitForAuth0AndCheck`, `checkAuthAndDisplayContent`. **Must load before `habits.js`.** |
| `/habits/habits.js` | Habits CRUD UI, wraps `updateContentVisibility` to call `loadTasks` after login. |

Site-wide scripts (Auth0 client, `handleLogin`, etc.) are assumed from the global layout/footer as on other course pages.

### `auth_page_load.js` — important symbols

| Function | Role |
|----------|------|
| `checkAuthAndDisplayContent()` | Resolves Auth0 client, optional redirect promise, then calls `updateContentVisibility(isAuthenticated)`. |
| `updateContentVisibility(isAuthenticated)` | Toggles `#private-content` vs `#login-required-message`. **Overridden** in `habits.js` to also run `loadTasks` when true. |
| `waitForAuth0AndCheck(maxRetries, retryDelay)` | Polls until `window.auth0` / `auth0Promise` exists, then runs auth check. |

### `habits.js` — important functions and globals

| Function / binding | Role |
|------------------|------|
| `loadTasks()` | GET `/api/habits`, filter by `user_id === email`, sort, render `#tasks-tbody`, toggle loading/error/empty/table. |
| `addTaskForm(event)` | POST new habit; clears inputs; `loadTasks()`. |
| `deleteTask(id)` | DELETE `/api/habits/:id` after confirm; `loadTasks()`. |
| `editTask(id)` | GET list, find row, populate `#editEventModal`, show modal. |
| `saveEditTask()` | PUT `/api/habits/:id`; hide modal; `loadTasks()`. |
| `escapeHtml(text)` | Sanitize habit name before `innerHTML`. |
| `formatDateShort(dateString)` | Display start date as `D-MMM-YY`. |
| `daysSince(dateString)` | Whole days from start to today (non-negative). |
| `rowActionHabitId` | Last clicked row’s id for row action modal (module-level `let`). |
| **`window.deleteTask` / `window.editTask` / `window.saveEditTask`** | Exposed for inline modal button and tooling. |
| **Wrapped `updateContentVisibility`** | Calls `originalUpdateContentVisibility`, then `setTimeout(loadTasks, 100)` if authenticated. |

**`DOMContentLoaded`**: attaches `#task-form` submit → `addTaskForm`; `#tasks-tbody` click → open `#rowActionModal`; `#row-action-edit-btn` / `#row-action-delete-btn` → `editTask` / `deleteTask`.

### Backend routes used

| Route | Method | Use |
|-------|--------|-----|
| `/api/habits` | GET | List habits (client filters by `user_id`). |
| `/api/habits` | POST | Create (`task`, `event_date`, `user_id`). |
| `/api/habits/:id` | PUT | Update (`task`, `event_date`, `user_id`). |
| `/api/habits/:id` | DELETE | Delete habit. |
